🔄 Der Perfekte Workflow

1. Initialisierung

Projekt setup mit npx claude-flow@alpha init --sparc --hive-mind. Erstellen Sie eine klare Projektstruktur und definieren Sie Namespaces.

2. Task Definition

Erstellen Sie eine TODO.md mit granularen Checkboxen. Jeder Task sollte objektiv messbar und in 15-30 Minuten abschließbar sein.

3. Swarm Activation

Starten Sie den Swarm mit passendem Preset: npx claude-flow@alpha swarm --preset enterprise --claude. Der Orchestrator verteilt Tasks automatisch.

4. Progress Monitoring

Überwachen Sie den Fortschritt mit hive-mind status. Bei Problemen greifen Sie mit spezifischen Agenten ein.

5. Validation & Testing

Führen Sie automatisierte Tests aus: sparc validate --comprehensive. Stellen Sie sicher, dass alle Erfolgs-Kriterien erfüllt sind.

6. Completion & Archival

Finalisieren Sie mit memory export --format complete. Erstellen Sie einen vollständigen Report und archivieren Sie das Projekt.

Pattern 2: Memory-Based Progress Tracking

Verwenden Sie Claude-Flow's Memory System für persistente Task-Verfolgung über Sessions hinweg.

progress-tracker.sh 
#!/bin/bash
# Task Progress Tracking with Memory

# Initialize tracking
npx claude-flow@alpha memory set \
  --namespace "auth-system" \
  --key "progress" \
  --value '{"phase": 1, "completed": [], "current": "setup"}'

# Execute task with continuation
npx claude-flow@alpha swarm "
1. Check memory for current progress
2. Read TODO.md
3. Continue from last checkpoint
4. Implement next uncompleted task
5. Update progress in memory
6. Commit changes
" --namespace "auth-system" --continue-session

# Check progress
npx claude-flow@alpha memory get \
  --namespace "auth-system" \
  --key "progress" --format json

Pattern 3: Automated Continuation Loop

Implementieren Sie einen automatischen Loop, der Tasks bis zur Fertigstellung fortsetzt.

auto-complete.js 
const { exec } = require('child_process');
const fs = require('fs');

async function autoCompleteTask() {
  const MAX_ITERATIONS = 10;
  let iteration = 0;
  
  while (iteration < MAX_ITERATIONS) {
    console.log(`🔄 Iteration ${iteration + 1}/${MAX_ITERATIONS}`);
    
    // Check TODO status
    const todoContent = fs.readFileSync('TODO.md', 'utf8');
    const incompleteTasks = todoContent.match(/- \[ \]/g)?.length || 0;
    
    if (incompleteTasks === 0) {
      console.log('✅ All tasks completed!');
      break;
    }
    
    console.log(`📋 ${incompleteTasks} tasks remaining`);
    
    // Execute next task
    await executeCommand(`
      npx claude-flow@alpha swarm "
        1. Read TODO.md
        2. Find next uncompleted task
        3. Implement it completely
        4. Mark as done in TODO.md
        5. Run tests if applicable
        6. Commit with descriptive message
      " --namespace project --continue-session --claude
    `);
    
    iteration++;
    
    // Brief pause between iterations
    await new Promise(resolve => setTimeout(resolve, 5000));
  }
  
  // Generate final report
  await executeCommand('npx claude-flow@alpha memory export --format report');
}

function executeCommand(cmd) {
  return new Promise((resolve, reject) => {
    exec(cmd, (error, stdout, stderr) => {
      if (error) {
        console.error(`Error: ${error}`);
        reject(error);
      } else {
        console.log(stdout);
        resolve(stdout);
      }
    });
  });
}

// Start auto-completion
autoCompleteTask().catch(console.error);

🔧 Häufige Probleme & Lösungen

❌ Claude vergisst vorherige Tasks

Symptome: Wiederholte Implementierungen, vergessene Features, inkonsistenter Code
Lösungen:
  • Verwenden Sie --continue-session Flag
  • Implementieren Sie Memory-basierte Checkpoints
  • Erstellen Sie eine COMPLETED.md Liste
  • Nutzen Sie namespace-basierte Isolation

❌ Context Window Overflow

Symptome: Abgeschnittene Responses, unvollständige Implementierungen, Fehler bei langen Files
Lösungen:
  • Teilen Sie Tasks in kleinere Chunks (max. 200 Zeilen)
  • Verwenden Sie Memory Storage für Zwischenergebnisse
  • Komprimieren Sie verbose Code
  • Nutzen Sie File-basierte Orchestrierung

❌ Inkonsistente Implementierungen

Symptome: Unterschiedliche Patterns, inkompatible Interfaces, Style-Inkonsistenzen
Lösungen:
  • Erstellen Sie eine STANDARDS.md mit klaren Guidelines
  • Verwenden Sie Linting und Formatting Tools
  • Implementieren Sie Review-Agenten
  • Nutzen Sie Template-basierte Generation
